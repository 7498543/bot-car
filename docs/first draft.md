## ESP32 智能小车软硬件开发需求

基于 ESP32 搭建四层分层软件架构，分内核、中间、运行、生态层；模块化硬件支持主板、屏幕、摄像头、电机、电源无缝升级兼容。实现小车遥控、循迹、避障基础功能，搭载 LVGL 人机交互界面；支持 WiFi / 蓝牙通信、NVS 参数存储、完整 OTA 远程升级，含固件校验、失败自动回滚防变砖。架构规避过度设计，轻量化适配小车场景，硬件更换无需大幅修改业务代码；输出架构、引脚、工程框架、HAL 接口、LVGL 界面五套交付物。

## 四层分层架构设计方案

1. 内核层：ESP-IDF+FreeRTOS 底层驱动、芯片资源、电源休眠管理，兼容 ESP32/S3；
2. 中间层：硬件抽象 HAL 统一外设接口，封装 OTA、存储、通信模块，实现硬件解耦；
3. 运行层：小车业务逻辑、LVGL 状态显示、OTA 执行任务；
4. 生态层：蓝牙 / WiFi 上位机控制、固件升级仓库。

硬件标准化引脚定义，分区表配置双 OTA 分区保障升级安全；整体极简轻量化，无冗余模块，外设、主控可迭代替换，兼顾扩展性与开发复杂度平衡。

## ESP-IDF 工程规范文档

工程分层目录：kernel 底层驱动、hal 硬件抽象层、middleware 中间服务、app 业务应用。

统一硬件 HAL 标准 API，电机 / 屏幕 / 电源 / 传感器调用接口固定；

搭载 LVGL 轻量化 UI 组件，实时展示电压、车速、升级进度。

内置 partitions.csv OTA 分区配置，sdkconfig 预设外设驱动开关；

代码低耦合，新增外设仅扩展 HAL 层，无需改动业务代码。

工程支持一键编译烧录，适配多版本 ESP32 主控，预留外设扩展接口，适配硬件迭代升级。
